# Current Clash95 HD State

## Thesis

Interpretation: the current Clash95 HD work has a passing 800x600 map evidence
baseline, while remaining work is concentrated in UI composition, castle
overview centering, and real/manual input validation rather than core 12x9 map
rendering. [source page: [[Current HD Map Evidence]]; source page:
[[Border And Bottom Tooltip Recovery Reports]]; source page:
[[Castle Barracks UI CDB Validation]]]

## Sourced Facts

- The current HD map smoke matrix reports overall `PASS`. [source page:
  [[Current HD Map Evidence]]]
- The patch-stage byte gate reports `118/118` selected current HD map bytes
  patched with no `original` or `unexpected` records. [source page:
  [[Current HD Map Evidence]]]
- Normal right/bottom blank cells in the post-owner evidence are classified as
  `visibility_zero`, and the forced-visible proof removes those blank active
  cells. [source page: [[Current HD Map Evidence]]]
- The border/tooltip reports say missing lower/right UI and bottom tooltip work
  is a UI composition/recovery problem, not a proven map-loop failure. [source
  page: [[Border And Bottom Tooltip Recovery Reports]]]
- Castle/barracks centered presentation and input paths have CDB proof for
  selected barracks action scenarios. [source page:
  [[Castle Barracks UI CDB Validation]]]

## Interpretation

- Treat [[Active HD Map Stage]] as the current map baseline until a new
  evidence index supersedes it.
- Treat [[Right-Bottom UI And Bottom Tooltip Recovery]] as the main remaining
  gameplay-screen UI problem.
- Treat [[Castle UI Centering State]] as promising but scoped: barracks/action
  UI paths have evidence, while full castle overview/interior dispatch still
  needs a narrow centering route.

## Uncertainty

- The exact owner of the ornate gameplay border and bottom tooltip/status band
  is still unresolved. [source page: [[Border And Bottom Tooltip Recovery Reports]]]
- The no-popup surface route is strong geometry evidence but not final
  authentic-color proof. [source page: [[Border And Bottom Tooltip Recovery Reports]]]
- Synthetic CDB input evidence does not fully replace manual DirectInput
  validation. [source page: [[Clash95 Engine Viewport Patch Notes]]]

## Contradictions

- No direct contradiction is recorded in this pass; the important clarification
  is that dark cells in the current save are visibility evidence, not current
  proof of a map rendering defect. [source page: [[Current HD Map Evidence]]]
